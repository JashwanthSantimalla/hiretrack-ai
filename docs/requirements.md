# Requirements

## Functional Requirements (Version 1)

### 1. Authentication

- User Sign Up
- User Login
- User Logout

---

### 2. Dashboard

The recruiter should be able to view:

- Total Job Openings
- Total Candidates
- Candidates Under Review
- Shortlisted Candidates
- Selected Candidates
- Rejected Candidates

---

### 3. Job Management

The recruiter should be able to:

- Create a Job Opening
- Edit Job Opening
- Delete Job Opening
- Paste a Job Description

---

### 4. Resume Management

The recruiter should be able to:

- Upload a single resume (PDF)
- Upload multiple resumes (PDF)
- Store uploaded resumes

---

### 5. Candidate Management

The system should automatically:

- Extract candidate information from resumes
- Create candidate profiles

The recruiter should be able to:

- View candidate profile
- Edit candidate details
- Add recruiter notes
- Update hiring status
- Delete candidate

---

### 6. Intelligent Candidate Matching

The system should:

- Analyze the Job Description
- Analyze uploaded resumes
- Compare resumes against the Job Description
- Calculate Candidate Match Score
- Identify missing skills
- Explain why a candidate matches the role
- Recommend suitable candidates

---

### 7. Search & Filters

Recruiters should be able to search and filter candidates by:

- Name
- Skills
- Experience
- Education
- Match Score
- Hiring Status

---

### 8. Recruitment Workflow

Candidates should move through stages:

- New Application
- Under Review
- Shortlisted
- Interview
- Selected
- Rejected

---

## Non-Functional Requirements

- Responsive Design
- Secure Authentication
- Fast Performance
- Clean & Modern UI
- Scalable Architecture
- Easy Navigation
